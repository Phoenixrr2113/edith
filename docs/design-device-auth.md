# Device Authentication Design

**Issue:** CLOUD-AUTH-048
**Status:** Implemented
**Blocks:** CLOUD-WS-047 (WebSocket protocol), CLOUD-FLY-043 (Fly.io deploy)

---

## Overview

When Edith runs on Fly.io, the WebSocket endpoint and any HTTP APIs must be authenticated so only Randy's devices can connect. This is a **single-user system** — no account management, no OAuth, no user database. A shared secret (`DEVICE_SECRET`) is generated once at setup and stored as a Fly.io secret and in the Tauri app's keychain.

---

## Approach: HMAC-signed JWT (HS256)

- **Algorithm:** HS256 (HMAC-SHA256) via Bun's built-in Web Crypto API — no external JWT library
- **Claims:** `{ sub: deviceId, deviceId, iat, exp }`
- **Lifetime:** 30 days
- **Refresh threshold:** client refreshes token on reconnect if expiry is within 7 days
- **Secret:** `DEVICE_SECRET` — 48 random bytes (hex), generated by `setup.sh`

---

## Token Lifecycle

```
┌─────────────┐        generateDeviceToken(deviceId, secret)
│ Setup / CLI │ ──────────────────────────────────────────────→  JWT (30d)
└─────────────┘

┌─────────────┐   Authorization: Bearer <jwt>
│ Tauri App   │ ──────────────────────────────────────────────→  WebSocket upgrade
└─────────────┘

┌─────────────┐        verifyDeviceToken(token, secret)
│  edith.ts   │ ──────────────────────────────────────────────→  { valid, payload, needsRefresh }
└─────────────┘

On reconnect (if needsRefresh):
  client calls refreshToken(oldToken, secret) → new JWT (30d)
```

## Device Registration

`DeviceRegistration` is a lightweight type for tracking known devices:

```typescript
interface DeviceRegistration {
  deviceId: string;       // e.g. "randy-macbook-pro"
  label: string;          // human-readable
  registeredAt: string;   // ISO timestamp
  lastSeenAt?: string;    // updated on each successful auth
}
```

In v1, device registration is implicit — any device with a valid token (signed with `DEVICE_SECRET`) is authorized. A future `packages/agent/.state/devices.json` registry can be added if per-device revocation is needed.

## Files

| File | Role |
|---|---|
| `lib/auth.ts` | `generateDeviceToken`, `verifyDeviceToken`, `refreshToken`, `extractBearerToken` |
| `lib/config.ts` | `DEVICE_SECRET = process.env.DEVICE_SECRET ?? ""` |
| `lib/cloud-transport.ts` | `authenticateUpgrade(req)` — called on every WebSocket upgrade |
| `setup.sh` | Auto-generates `DEVICE_SECRET` with `openssl rand -hex 48` |

## Security Notes

- `DEVICE_SECRET` must never be logged or returned in API responses
- Store as `fly secrets set DEVICE_SECRET=<value>` on Fly.io
- Store in Tauri app's OS keychain (not plain localStorage)
- HTTP endpoints in `edith.ts` should also check the Bearer token before responding
- If `DEVICE_SECRET` is not set, all tokens are rejected with a console error

## WebSocket Upgrade Flow

```
Client                                    edith.ts (Bun.serve)
  │── GET /ws  Authorization: Bearer jwt ──→│
  │                                         │  authenticateUpgrade(req)
  │                                         │    extractBearerToken(header)
  │                                         │    verifyDeviceToken(token, DEVICE_SECRET)
  │                                         │      HS256 verify → claims → exp check
  │←── 401 Unauthorized ───────────────────│  (if invalid/expired)
  │
  │←── 101 Switching Protocols ────────────│  (if valid)
  │                                         │  server.upgrade(req, { data: { deviceId } })
  │←── { type: 'connected', deviceId } ────│
```

## Refresh Flow

```
// On reconnect, client checks token expiry before upgrading:
const result = await verifyDeviceToken(storedToken, secret);
if (!result.valid) {
  // expired/invalid — cannot refresh, must re-provision from DEVICE_SECRET
}
if (result.needsRefresh) {
  const newToken = await refreshToken(storedToken, secret);
  // store newToken in keychain, use for this connection
}
```

`refreshToken()` only works if the current token is still valid (not yet expired). Expired tokens cannot be refreshed — a new token must be generated from `DEVICE_SECRET`.

## Dependencies

None (uses Bun's built-in Web Crypto API). No `jose`, `jsonwebtoken`, or other JWT packages required.
